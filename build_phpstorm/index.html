phpstorm配置

同时构建docker + 多版本php + xdebug远程开发的雄心壮志#

首先去clone一个 dnmp 项目，虽然原本的项目长期不更新了，但是还有别人替作者填坑 比如这个

此处我是从码云上克隆的，无关紧要

cd 到./dnmp/dnmp 目录下，可以看到作者留下的配置文件模板

可以看到作者贴心地把仓库镜像换成了国内的地址，所以不用担心需要下载很多php内容的问题

然后按项目文档说的那样，把配置文件模板拷贝成配置文件

cp env.example .env

一般情况下我的nginx都是映射到80端口的，所以不需要修改什么配置文件的内容

但是因为docker-compose的版本比较低，所以还需要把docker-compose.yml里面的版本从3改为2才能运行

完成以上操作之后使用

docker-compose up -d 构建完毕以后访问localhost就能看到phpinfo了

往下翻可以看见有xdebug存在，不过默认是没有配置IDEkey的，所以我们还要修改一下php.ini的内容

外部的php.ini在dnmp目录的conf下，修改如下内容：

[XDebug]
xdebug.remote_enable = 1
xdebug.remote_handler = dbgp
xdebug.remote_connect_back=1
; Set to host.docker.internal on Mac and Windows, otherwise, set to host real ip
xdebug.remote_host = host.docker.internal
xdebug.remote_port = 9001
xdebug.remote_log = /var/log/php/xdebug.log
xdebug.idekey = PHPSTORM

为了防止引起冲突，我把debug端口改成9001了， xdebug.remote_host = host.docker.internal 这样设定可以让容器内部的ip直接指向宿主机的ip，这是docker18.03新增加的特性

修改完毕以后回到docker-compose的目录，把整个项目重启一遍，配置文件就能奏效了

然后，为了让xdebug顺利运作起来，我们还需要让云服务器和主机处于同一个网段

这里我们用zero-tier实现这个功能，先去官网注册一个账号，在network里生成一个自己的节点

然后在自己电脑上和云服务器上下载zero-tier，指令如下

curl -s https://install.zerotier.com/ | sudo bash

下载以后发现官网并没有给我们直接建立软链接，我们就直接进入安装路径手动启动然后加入节点

云服务器和主机都加入节点以后，在zero-tier管理界面把两个用户认证都给放通

相互之间就能ping通了，主机直接访问云服务器的内网ip也能看到phpinfo，这样再建立起xdebug就能回显到本地了

回到phpstorm，新建项目

再按照图示建立起远程连接来，因为跟云服务器已经在一个子网内了，所以可以直接使用zero-tier给的ip建立远程连接

设置完毕以后可以把箭头下面的Automatic upload给勾上，然后项目文件就会自动地上传到云服务器上了

编码格式根据云服务器自身的格式来，我的服务器是utf-8的，所以要修改为utf-8

mappings设置云服务器上对应的站点路径，在dnmp这个项目中，站点的路径默认在dnmp/dnmp/www/site/，所以设置到对应的这个地方就行了

然后在右边添加一个远程debug的配置

记得也要把IDEkey的选项勾上，方便之后在浏览器里面通过xdebug插件进行调试

把右边配置好以后，我们新建一个php文件，再随便写点啥，发现它已经会自动上传到云服务器上了

最后在这个地方配置xdebug

此处必须要保证第二步全部都要出现绿色的勾才行，我这里是因为xdebug端口没有和远程环境的一致导致的，去 File | Settings | Languages & Frameworks | PHP | Debug 里面修改为9001就能成功运行

完事后我们打上断点，然后直接在浏览器里运行，xdebug就会自行启动了

这个需要配合chrome的xdebug插件实现，自行去商店下载，然后把ide改成phpstorm就可以了，别的地方不需要修改

如果出现问题，可以试着修改一下 File | Settings | Languages & Frameworks | PHP | Servers 里面的mapping选项，左边对应主机里面的项目路径，右边对应docker中的web目录
