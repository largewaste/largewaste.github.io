hexo的折腾记录

step.1 安装nodejs和npm#

这里还是使用网上推荐的方法而不是直接sudo apt-get install nodejs

因为这里可能由于版本等各种各样的原因导致node和npm无法正常使用，

所以直接去官网下载解压然后建立软连接的方法

操作如下：

wget http://nodejs.org/dist/latest/node-v10.11.0-linux-x64.tar.gz #可根据具体需求下载不一样的版本，里面会自动集成node跟npm
tar xf node-v5.10.1-linux-x64.tar.gz -C /usr/local/
cd /usr/local/
mv node-v5.10.1-linux-x64/ nodejs #这一步是把解压出来的内容放到以nodejs命名的文件夹下，方便之后的操作
ln -s /usr/local/nodejs/bin/node /usr/local/bin
ln -s /usr/local/nodejs/bin/npm /usr/local/bin
#这两步是单独把npm跟node与环境变量建立软连接，这样就可以直接在shell里使用这两种指令了

如此这般操作之后，npm和node就能使用了

step.2 安装hexo#

有了npm以后就可以直接使用npm的源安装hexo了，操作如下：

npm install -g hexo-cli

安装之后再像刚才那样建立一遍软连接，就可以正常使用了

ln -s /usr/local/nodejs/bin/hexo /usr/local/bin

或者直接把node的路径添加到环境变量中，一劳永逸

export PATH=$PATH:/usr/local/nodejs/bin

step.3 初始化#

安装完成后就可以自己找一个目录建立一个hexo的运行路径了，像下面这样操作就能顺利让网站运行起来了

hexo init largewaste #这里写目录名称
cd largewaste
hexo g
hexo c

这样，我们就可以看到hexo已经在4000端口成功运行起来了

step.4 设置反向代理#

因为hexo的页面默认是在4000端口的，所以要设置一下反向代理把4000端口的内容转发到80端口上

于是我们可以在/etc/nginx/conf.d这个目录下新建一个配置文件，文件保证是.conf格式的就行

然后我们在里面添加如下内容就行

upstream magicaltapir {
server 127.0.0.1:4000;
keepalive 64;
}
server {
listen 80;
server_name www.magicaltapir.xyz magicaltapir.xyz;
location / {
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header Host $http_host;
proxy_set_header X-Nginx-Proxy true;
proxy_set_header Connection "";
proxy_pass http://magicaltapir;
}
}

保存以后再重启nginx就可以生效了

以上只是反向代理最简单的设置方法，更详细的就不再介绍了

step.5 设置防火墙#

我这边使用的Debian系统是不会自动配置防火墙的，需要使用的iptables又过于麻烦，于是干脆就直接下个ufw来用，比较省事

sudo apt-get install ufw
sudo ufw allow 22 #在开启ufw一定要记得先把ssh的端口给放通了，不然到时候会把所有端口都关闭
sudo ufw allow 80
sudo ufw enable

这样一来除了22和80端口以外，其他所有的端口都会处于关闭状态了

step.6 完事开搞#

剩下的工作比如说更换主题就不用再说了吧
